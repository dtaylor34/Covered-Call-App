Overview

API Playbook

The APIs tab lets users connect a real brokerage account to the Covered Calls Manager. Once connected, the app stops using static data and starts pulling live quotes, real option chains, and actual account positions — turning the app from an educational simulator into a live companion for managing real covered call positions.

This document explains what the API tab does, what it enables in every part of the app, what its limits are, and exactly what a user needs to do to set it up.

ℹ
Manual mode stays fully functional. The API is additive — connecting a broker account enhances the app with live data, but every feature works without it. Users who don't connect a broker continue using the app exactly as before.

User journey

Getting started

The one-time setup takes about 10 minutes plus a 2–5 day wait for Schwab's approval. After that, every session reconnects automatically.

1
Open the APIs tab in the app
Scroll the tab bar to find the 🔌 APIs tab. You'll see the broker hub with Charles Schwab available and other brokers listed as coming soon.
2
Click "Connect" on Charles Schwab
The Schwab setup flow opens. You'll see the step-by-step guide and the exact callback URL to paste into Schwab's Developer Portal.
3
Create a Schwab Developer account and register an app
Go to developer.schwab.com — this is a separate account from your brokerage login. Register an app, select both API products, paste in the callback URL the app shows you.
4
Wait for Schwab approval (2–5 business days)
Schwab reviews all developer app registrations. Status shows "Approved — Pending" initially. You'll get an email when approved.
5
Paste your App Key and App Secret into the app
Copy both from your approved Schwab Developer app. These are stored encrypted — the app never exposes them to the browser or logs them anywhere.
6
Click "Connect Schwab Account" and complete OAuth
A Schwab login window opens. Sign in with your regular brokerage credentials (not your developer account), then select which accounts to authorize.
7
Select your default account
After OAuth completes, the app shows all linked accounts (IRA, individual, etc.) with masked account numbers. Pick your default — you can switch any time from the APIs tab.
8
Live data starts flowing
The status dot in the APIs tab turns green. Real-time quotes, positions, and option chains are now active across the app.

Accounts

Switching between accounts

Most users have multiple accounts — Individual, Roth IRA, Traditional IRA. The app lets you connect all of them in one OAuth flow and switch between them without reconnecting.

How account switching works

  • During the OAuth flow, Schwab shows all accounts in your brokerage. You authorize access to all of them at once.
  • The APIs tab shows all linked accounts with masked numbers (e.g. ····1234 ····5678).
  • Tap any account to make it the default. The selection persists across sessions.
  • Switching accounts re-fetches positions and buying power for that account. Quotes and option chains are account-independent — they switch instantly.
  • The active account is shown in the live data banner at the top of the APIs tab.
ℹ
The app only reads account data — positions, balances. It does not place trades. Order entry is not part of Phase 1 (see Not supported yet).
Authentication

Token lifecycle & reconnecting

Schwab uses a two-token system. Understanding it explains why you occasionally need to re-authenticate.

ACCESS TOKEN
Expires every 30 min
Used for every API call. Refreshed automatically by the app in the background — you never notice this.
REFRESH TOKEN
Expires every 7 days
Used to get new access tokens. The app refreshes this proactively. If it does expire, you'll be prompted to re-authenticate once.

Status indicators in the APIs tab

Status dotLabelWhat it meansAction
Connected Tokens valid, live data flowing None needed
Refreshing Access token being renewed silently None — wait a moment
Expired Refresh token expired (7-day limit) Click "Reconnect" in APIs tab
Disconnected No broker connected Go to APIs tab → Connect
What gets unlocked

Live quotes

Without the API, stock prices come from Yahoo Finance with a 15-minute delay. With the Schwab API connected, prices are real-time (NBBO — National Best Bid and Offer).

Where live quotes appear

LocationWithout APIWith API
Dashboard tab — quote card 15min delayed Real-time NBBO
Working tab — stock price in P&L table Static / manual input Live, auto-refreshed
Working tab — urgency signal Based on delayed price Based on live price
Selection tab — B-S calculator Static stock data Live underlying price
Risk tab — strike distance Manual entry Calculated from live price

Quotes refresh automatically every 30 seconds while the app is in focus. They use Schwab's batch quote endpoint — multiple tickers in one call — so the 120 req/min rate limit is rarely a concern.

What gets unlocked

Auto-imported positions

The most significant quality-of-life improvement. Instead of manually logging positions in the Trades tab, the app reads your actual open positions from Schwab.

How it works

  • On app load: Positions are fetched from Schwab for the active account and reconciled with the transaction log.
  • What's imported: Open short call positions (covered calls you've sold). The underlying stock holdings are also visible for context.
  • Matching: Imported positions are matched to existing manual log entries by symbol + strike + expiration. If matched, the manual entry is updated. If new, it's added automatically.
  • Manual entries preserved: Any positions you logged manually that don't exist in Schwab are kept as-is (they may be in a different account or already closed).
  • Refresh: Positions sync on load and via the manual refresh button. They do not auto-poll (avoids rate limit waste).
ℹ
Position sync is read-only. The app cannot place, modify, or cancel orders. This is intentional for Phase 1 — order routing is planned for Phase 5 as an opt-in feature with an explicit review screen.

Limitations

Historical data gap

This is the most important limitation to understand. Broker APIs — including Schwab — do not provide historical options pricing data. You can see today's option chain, but not what premiums were for a given strike three months ago.

How the app handles this

Data typeSourceAvailable?

Historical stock price (OHLCV) Yahoo Finance ✓ Yes — years of data
Historical IV / HV Yahoo Finance Partial — near-term only
Historical option chain snapshots CBOE free data Index options only (SPX, VIX)
Historical option pricing (individual equities) Polygon.io free tier EOD, 5 calls/min limit

The educational backtesting features in the app already use Yahoo Finance and CBOE data. Connecting a broker API does not change this — those tabs work exactly the same before and after connecting.

Troubleshooting

"Connect Schwab Account" button does nothing / window doesn't open

Your browser is likely blocking the popup. Allow popups for the app's domain in your browser settings, then try again. Safari and Firefox can be especially aggressive about blocking OAuth windows.

OAuth completes but the app shows "Pending OAuth" instead of "Connected"

The callback page (/api/schwab/callback) may have been blocked or the tab closed before the token exchange finished. Go to APIs tab → Schwab → the setup flow will detect the pending state and let you re-attempt the OAuth step without re-entering your credentials.

"Token exchange failed" error

This usually means the App Key or App Secret was entered incorrectly. Return to the Schwab Developer Portal, copy both values fresh, and re-enter them. Also confirm your app status in the portal is "Approved" (not "Approved — Pending").

Status shows "Connected" but quotes are not updating

Check the Connection Health section in the APIs tab. If the access token shows "Expired," click the manual refresh button. If the refresh token is expired, you'll need to reconnect (re-run the OAuth flow — you don't need to re-enter your App Key/Secret).

I see accounts I didn't expect / accounts are missing

The accounts shown are exactly what Schwab returned during your OAuth grant. If you have multiple Schwab logins (e.g. a personal and a business account), make sure you logged in with the correct one. You can disconnect and reconnect to switch the authorized account set.

The APIs tab appears but shows no broker options

This is likely a Cloud Functions deploy issue. Open the browser console and look for a Firebase function error. Run firebase deploy --only functions to make sure the Schwab functions are deployed, then reload the app.

FAQ

Can the app place trades on my behalf?

No. Phase 1 is read-only. The app reads your positions and market data but cannot submit, modify, or cancel orders. Order routing is planned for Phase 5 as an explicit opt-in feature.

Where are my App Key and App Secret stored?

Encrypted in Firestore under your user document. They are only accessed by Cloud Functions running server-side — they never appear in browser logs, network requests from the frontend, or any client-readable location.

Can I connect multiple brokers at the same time?

Yes — you can have connections for multiple brokers simultaneously (as more brokers are added in Phase 2+). However, only one broker is "active" at a time as the live data source. You toggle which is active from the APIs tab.

Do I need ThinkOrSwim enabled on my Schwab account?

Yes, technically — the Schwab API requires TOS to be enabled to access trading endpoints. Most former TD Ameritrade accounts and active traders already have it. If not, you can enable it in your Schwab account settings or by calling Schwab.

What happens if I disconnect a broker?

The API connection and tokens are removed from Firestore. The app reverts to manual mode — Yahoo Finance delayed data and manual position logging. Your transaction log history is preserved, nothing is deleted.

Does the API work outside US market hours?

Quotes can be fetched outside market hours — you'll see the last traded price. Option chains outside hours show the last bid/ask, which may be stale. Position and account data is always available. The app shows an "as of close" label when market is closed.

How is this different from linking Schwab in PlaidLink or a similar service?

This is a direct integration with Schwab's official developer API — not a screen-scraper or read-only account aggregator. It uses Schwab's own OAuth and gives access to live market data (quotes, option chains) which aggregators don't provide. The trade-off is the 2–5 day developer app approval process.
